# ADR-002: 픽셀 스타일이 아닌 현대적 UI

## 상태

승인됨 — M0

## 배경

제품은 픽셀 아트를 다루지만 일반 메뉴·패널·설정까지 픽셀 스타일로 만들면 가독성, 접근성, 고밀도 데스크톱 작업 흐름을 해친다. 특정 기존 편집기의 외형을 복제해서도 안 된다.

## 결정

픽셀 정확성은 향후 캔버스와 결과물에만 적용한다. M0 셸은 시스템 폰트, SVG 벡터 아이콘, CSS Custom Properties, 명확한 패널 경계와 작은 곡률을 사용한다. System/Dark/Light, 80%–200% UI 배율, `prefers-reduced-motion`을 지원한다.

## 이유

시스템 폰트와 벡터 아이콘은 여러 배율과 운영체제에서 선명하다. 디자인 토큰은 다크·라이트 대비를 일관되게 유지한다. 정보 밀도가 높은 직사각형 작업 공간은 제작 도구의 패널 구조와 맞는다.

## 결과

- UI는 픽셀 폰트, 비트맵 아이콘, 계단형 테두리를 사용하지 않는다.
- 테마와 배율이 모든 셸 요소에 일관되게 적용된다.
- 향후 픽셀 캔버스는 일반 UI와 별도의 렌더링·정확성 규칙을 가져야 한다.
- 시각 변경 시 다크·라이트와 고배율 접근성을 함께 검증해야 한다.

## 검토한 대안

- 전체 픽셀 아트 UI: 제품 주제와 직관적이지만 가독성과 접근성이 낮다.
- Aseprite 외형 복제: 친숙할 수 있으나 독자 제품 정체성과 요구사항에 맞지 않는다.
- 모바일 카드 중심 UI: 데스크톱 제작 도구에 필요한 정보 밀도를 제공하지 못한다.
