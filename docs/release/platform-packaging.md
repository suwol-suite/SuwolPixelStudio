# Platform packaging

CI creates unsigned package smoke builds on Windows x64, macOS, and Linux x64 without release secrets. Tag releases start two independent workflows. `release-core.yml` builds Windows and Linux in parallel, publishes their assets with an initial signed `checksums.txt`, and leaves the Release public. `release-macos.yml` signs and notarizes arm64 assets in parallel, waits at most ten minutes for that Release, uploads to it, downloads the complete asset set, and replaces the checksum and signature with a deterministic five-asset version.

Windows intentionally has no Squirrel, Setup.exe, NUPKG, MSI, code signing, or automatic replacement. Its unsigned executable and BrowserWindow use the multi-resolution `apps/desktop/assets/icon.ico`. Linux emits x64 ZIP and AppImage; Forge and BrowserWindow use `apps/desktop/assets/linux/studio.suwol.pixel.png`, and the ZIP also contains desktop/MIME resources, LICENSE, and notices. AppImage creation uses the Forge-compatible ReForged maker and `squashfs-tools`, with `SOURCE_DATE_EPOCH` set from the release tag commit. The type-2 runtime is downloaded from the official AppImage Release endpoint and must match the reviewed SHA-256 embedded in `release-core.yml`; an upstream runtime change fails closed until that hash is reviewed. macOS Forge, the app bundle, document UTI, and DMG all reference `apps/desktop/assets/icon.icns`; BrowserWindow does not override the bundle icon.

macOS exports the `studio.suwol.pixel.document` UTI for `.suwolpixel` and uses the application ICNS as its document icon. Linux carries both `studio.suwol.pixel.png` and `application-x-suwol-pixel-studio.png` for desktop and MIME integration. Portable ZIP extraction alone does not register file associations: Linux users or distributors must install the desktop, MIME, and icon resources, while Windows intentionally has no `.suwolpixel` shell association because M6 forbids an installer and registry mutation.

Canonical asset names are `SuwolPixelStudio-<version>-win-x64.zip`, `SuwolPixelStudio-<version>-linux-x64.zip`, `SuwolPixelStudio-<version>-linux-x64.AppImage`, `SuwolPixelStudio-<version>-mac-arm64.zip`, and `SuwolPixelStudio-<version>-mac-arm64.dmg`. `checksums.txt` includes only these distribution assets, sorted by filename; it never includes itself or `checksums.txt.asc`.

Run `pnpm make --arch=<arch>`, `pnpm release:prepare --platform=<win|linux|mac> --arch=<arch>`, and `pnpm release:validate --scope=<win|linux|mac>`. Core/final checksum generation uses `pnpm release:checksums --scope=<core|all>` followed by the same command with `--verify`. A release tag must first pass `pnpm release:verify-tag --tag=v<version>`.
